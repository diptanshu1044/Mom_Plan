# MCP Integration Guide

The MomPlan project heavily leverages the Model Context Protocol (MCP) to streamline development and production processes.

## 1. Stitch UI MCP
We use the **Stitch UI MCP** server to rapidly prototype, update, and manage the frontend design system without writing raw CSS from scratch.

### How it is used:
- **Design System Generation**: The base UI for MomPlan (Project ID: `6722713828724750310`) was established using `mcp_StitchMCP_create_design_system`.
- **Screen Updates**: When counselors or administrators request new dashboards (e.g., for analytics or advanced search), developers can use `mcp_StitchMCP_edit_screens` to seamlessly update the UI.
- **Component Consistency**: The `frontend_admin/` uses the CSS output generated by Stitch, ensuring that the "Polished Startup / Glassmorphism" style remains consistent.

## 2. Neon Database MCP
We use the **Neon MCP** server (`mcp-server-neon`) to automate database branching, migrations, and performance tuning.

### How it is used:
- **Zero-Downtime Migrations**: When adding new federal or state program schemas, developers can use `mcp_mcp-server-neon_prepare_database_migration` to test changes in isolated branches before applying them to production.
- **Branching**: For local development, every developer can spawn a new branch of the Neon database using `mcp_mcp-server-neon_create_branch`, ensuring they don't corrupt the staging or production data.
- **Query Tuning**: As the `eligibility_results` table grows, `mcp_mcp-server-neon_prepare_query_tuning` is used to suggest composite indexes for high-volume analytics queries used by the Admin Portal.
